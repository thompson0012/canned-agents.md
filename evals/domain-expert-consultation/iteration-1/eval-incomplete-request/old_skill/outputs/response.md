## General Implementation Advice

### 1. Define Clear Requirements
Before writing any code or choosing any tools, make sure you have a solid understanding of:
- **What problem** the system is solving
- **Who the users** are (internal team, customers, both?)
- **What success looks like** — measurable outcomes

### 2. Start with Architecture
- Sketch out the high-level components and how they interact
- Identify data flows, storage needs, and integration points
- Consider scalability, security, and maintainability from the start

### 3. Choose the Right Technology Stack
- Match your stack to your team's expertise and the problem domain
- Avoid over-engineering — pick tools that fit your scale today with a reasonable growth path
- Consider ecosystem maturity, community support, and long-term viability

### 4. Plan for Iterative Delivery
- Break the implementation into phases or milestones
- Deliver a minimal viable version first, then iterate
- Build in feedback loops with stakeholders at each milestone

### 5. Don't Skip the Fundamentals
- Set up version control, CI/CD, and automated testing early
- Establish coding standards and documentation practices
- Plan for monitoring, logging, and error handling from day one

## Clarifying Questions

To give you more specific and useful advice, I'd need to understand:

1. **What type of system?** (e.g., web application, API/backend service, data pipeline, mobile app, internal tool, infrastructure/platform)
2. **What is the scale?** (expected users, data volume, request throughput)
3. **What is your team composition?** (solo developer, small team, large organization — and what tech expertise exists?)
4. **What is your timeline?** (proof of concept in a week, production system in 6 months, etc.)
5. **Are there existing systems this needs to integrate with?** (databases, APIs, third-party services)
6. **What are your constraints?** (budget, hosting environment, compliance/regulatory requirements, legacy compatibility)
7. **Is this greenfield or a replacement/migration?** (building from scratch vs. replacing something that exists)
8. **What does "implementing" mean to you right now?** (Are you at the planning stage, technology selection, or ready to start coding?)
